Claude Code /commands

Управление контекстом, памятью и сессиями для вайб-кодинга

Быстрый старт: введи / в пустой строке — покажет все доступные команды. Фильтрует по мере набора.
Категории: ● Контекст ● Память ● Сессия ● Модель ● Код и ревью
🪟 Контекст — самое важное
/context
Показать заполненность контекстного окна. Цветная сетка — сколько занято файлами, историей, системным промптом. Твой «датчик топлива».
/compact
Сжать историю, сохранив суть. Суммирует диалог — контекст уменьшается, важное остаётся. Запускай при 70–80% заполнения, не жди лимита.
/compact инструкция
Компакт с указанием что сохранить. Например: /compact keep auth flow and DB schema — оставит именно эти детали, остальное сожмёт.
/clear
Полный сброс истории. Контекст пуст — как новая сессия. Файлы, которые Claude редактировал, остаются. Используй при переходе к новой задаче.
/cost
Стоимость и токены текущей сессии. Показывает input/output токены и $ потрачено. Следи во время длинных агентных циклов.
/usage
Лимиты и rate limit. Проверь, не близко ли к порогу запросов.
Правило: /compact — при работе над той же задачей. /clear — при переходе к другой. Старый контекст мешает больше, чем помогает.
🧠 Память — персистентный контекст
CLAUDE.md Память проекта
./CLAUDE.md
Claude читает этот файл в начале каждой сессии. Пиши сюда: стек, правила кода, архитектуру, что нельзя трогать. Создаётся через /init или вручную.
~/.claude/CLAUDE.md Глобальная память
~/.claude/CLAUDE.md
Твои персональные правила для всех проектов: стиль кода, предпочтения, запреты. Работает везде.
/memory Редактор памяти
/memory
Открывает CLAUDE.md прямо из сессии для редактирования. Добавляй заметки на лету — они останутся в следующей сессии.
# shortcut Быстрая запись в память
# текст
Начни сообщение с # — Claude сохранит это как заметку в память без добавления в основной диалог.
Что писать в CLAUDE.md: tech stack, соглашения по именованию, структуру папок, что не трогать (prod DB, env-файлы), линтеры и форматтеры, команды запуска тестов.
💬 Управление сессиями
claude -c
Продолжить последнюю сессию (из терминала, до запуска). Идеально чтобы вернуться к вчерашнему контексту.
claude -r "session-id"
Возобновить конкретную сессию по ID. ID показывается при старте или через /status.
/status
Версия, session ID, статус подключения. Нужен ID — смотри здесь.
/todos
Список задач из текущей сессии. Claude автоматически трекает таски упомянутые в диалоге.
/btw вопрос
Задать вопрос «в сторону» не прерывая основную задачу. Контекст не затрагивается. Работает пока Claude что-то делает.
/branch
Создать ответвление разговора. Попробуй альтернативный подход не теряя текущий контекст.
🤖 Модель и режимы
/model
Сменить модель прямо в сессии без перезапуска. Список выбора появится интерактивно.
/model sonnet
Sonnet 4.6 — дефолт для большинства задач. Баланс скорости и качества. Используй 80% времени.
/model opus
Opus 4.6 — для сложных задач. Архитектурные решения, сложный баг, рефакторинг большой кодовой базы. Дороже — переключайся обратно после.
/model haiku
Haiku 4.5 — быстро и дёшево. Простые правки, форматирование, переименование, шаблонный код.
/effort
Уровень «размышления» Claude. Интерактивный слайдер. Больше effort → глубже анализ → медленнее и дороже.
🔍 Код, ревью и разработка
/init
Создать CLAUDE.md для проекта. Запускай в новом проекте первым делом — Claude изучит структуру и сам заполнит файл.
/review
Код-ревью последних изменений. Структурированный анализ: баги, стиль, безопасность. Надёжнее чем «посмотри мой код» в чат.
/diff
Показать текущий diff прямо в Claude. Все изменения сессии — без выхода в git.
/plan задача
Составить план перед работой. Например: /plan refactor auth without breaking tests. Хорошая практика перед сложными правками.
/debug проблема
Целенаправленная отладка. Claude трейсит поток выполнения, ищет причину. Работает лучше с конкретным описанием.
! команда
Запустить shell прямо из чата. ! git status, ! npm test — вывод попадает в контекст. Без переключения между вкладками.
@файл.ts
Включить файл в промпт. @src/auth.ts — Claude видит содержимое. Работает с папками: @src/components/.
⚡ Кастомные команды
.claude/commands/
Команды проекта — доступны всей команде, коммитятся в репо. Создай файл review.md → вызывай через /review.
~/.claude/commands/
Персональные команды — работают во всех проектах. Твои личные шорткаты.
$ARGUMENTS
Передать аргументы в команду. В файле команды напиши $ARGUMENTS — при вызове /fix-bug 123 high подставятся значения.
Лайфхак: попроси Claude создать команду за тебя — «создай slash-команду /ship которая делает review, коммит и пуш». Он сам напишет markdown с нужным front matter.
🔄 Рабочие сценарии для вайб-кодинга
▸ Типичные последовательности команд
01 · Начало нового проекта
/init /memory /context
Создай CLAUDE.md, добавь правила проекта, проверь что контекст загружен правильно.
02 · Длинная фича-сессия
/context /compact keep auth logic /cost
Каждые 20–30 мин проверяй контекст. Компакти с фокусом. Следи за ценой.
03 · Перед мёрджем в main
/review /diff ! git commit -m "..."
Ревью, сверь дифф, закоммить. Claude сам пишет сообщение коммита если попросить.
04 · Переключение задач
/clear # новый контекст
Сначала очисти всё, потом коротко напомни Claude что сейчас важно через # заметку.
05 · Вернуться на следующий день
claude -c /todos /context
Продолжи последнюю сессию, глянь список задач, проверь заполненность контекста.
06 · Сложная архитектурная задача
/model opus /plan рефакторинг /model sonnet
Переключись на Opus для планирования, вернись на Sonnet для выполнения.
