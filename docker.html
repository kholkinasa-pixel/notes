Git и деплой независимы. Можно задеплоить без коммита и закоммитить без деплоя. Коммит = «сохранил версию в историю». Деплой = «пересобрал и перезапустил бота из текущих файлов».
📦

Docker простыми словами: контейнер — это «коробка», внутри которой крутится бот в собственном мини‑Linux. Меняешь код — коробку надо пересобрать, старый процесс внутри сам не обновится.

1

Подключиться через VS Code

  • Поставь расширение Remote - SSH (от Microsoft), если ещё нет.
  • Внизу слева синяя кнопка >< → Connect to Host → выбери hetzner.
  • Open Folder → /root/projects/edit-bot.
  • Терминал на сервере: меню Terminal → New Terminal (он уже внутри сервера).
✳️

Теперь ты редактируешь файлы прямо на сервере — те же, из которых собирается бот. Rsync больше не нужен.

2

Проверить, что бот поднят

Работает из любой папки:

docker ps

Ищешь строку projects-edit-bot-1 и смотришь колонку STATUS:

Up …бот работает.
Exitedупал — смотри логи (шаг 3).
3

Посмотреть логи

docker logs -f projects-edit-bot-1   # следить вживую, Ctrl+C — выйти
docker logs --tail 50 projects-edit-bot-1   # последние 50 строк

-f = новые строки появляются сразу. Ctrl+C закроет только просмотр логов, не бота.

Что значат типичные строки:

всё ок Run polling for bot @editeditorBot + Start polling — бот слушает Telegram.
конфликт TelegramConflictError … terminated by other getUpdates — запущена вторая копия бота (например, случайно bot.py на ноутбуке). Выключи её.
⚠️

Два экземпляра с одним токеном не уживаются — всегда должна работать только одна копия.

4

Задеплоить изменения

После правок кода — из папки /root/projects (там лежит docker-compose.yml):

cd /root/projects
docker compose up -d --build edit-bot
  • --build — пересобрать коробку с новым кодом. Без этого запустится старая версия!
  • -d — в фоне.
  • edit-bot — трогаем только этого бота, habit‑tracker не задеваем.

После деплоя проверь логи (шаг 3), что поднялся без ошибок.

docker compose restart edit-bot   # перезапуск без пересборки (напр. меняла .env)
docker compose stop edit-bot      # остановить
docker compose up -d edit-bot     # снова запустить
🔒

Токен .env и реестр каналов channels.json переживают пересборку — трогать их при деплое не надо.

5

Коммитить в git

Через кнопки VS Code (проще): панель Source Control (иконка с ветками) → видишь изменённые файлы → пишешь сообщение коммита вверху → ✓ Commit → Sync / Push.

Через терминал:

git add .
git commit -m "что поменяла"
git push
🛡️

.env, channels.json, *.log в гит не попадут — они в .gitignore, токен в репозиторий не утечёт.

🐳

Панель Docker в VS Code (иконка-кит): контейнер projects-edit-bot-1 → правый клик → View Logs / Restart / Stop мышкой, без команд. Зелёный значок = работает.
